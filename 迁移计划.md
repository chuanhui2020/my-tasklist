# My-Tasklist 迁移计划

## 目标

- 前端迁移到 Cloudflare Pages（免费托管）
- 后端 + MySQL 合并到另一台 1C2G 新加坡服务器（已运行 kiro-gateway、sub2api 等服务）
- 退掉当前 LightNode 服务器，每月省 $7

## 当前架构

```
LightNode 1C2G 新加坡（$7/月）
├── tasklist-frontend   (Nginx: 静态文件 + API反代 + Grafana反代 + SSL)
├── tasklist-backend    (FastAPI + Uvicorn)
├── tasklist-db         (MySQL 8.4)
├── tasklist-prometheus
├── tasklist-grafana
└── tasklist-mysql-exporter

域名：
- tasklist.ch-tools.org → 前端 + API（通过 Nginx 反代）
- grafana.ch-tools.org → Grafana（通过 Nginx 反代）
- SSL: Cloudflare Origin 证书
```

```
另一台 1C2G 新加坡服务器
├── kiro-gateway        (~100MB)
├── sub2api             (~19MB)
├── sub2api-postgres    (~41MB)
└── sub2api-redis       (~5MB)
总内存占用: ~164MB / 1.93GB，剩余充足
```

## 迁移后架构

```
Cloudflare Pages（免费）
└── tasklist 前端静态文件（Vue 3 SPA）
    域名: tasklist.ch-tools.org

另一台 1C2G 新加坡服务器
├── kiro-gateway        (已有)
├── sub2api             (已有)
├── sub2api-postgres    (已有)
├── sub2api-redis       (已有)
├── tasklist-backend    (新增, FastAPI)
├── tasklist-db         (新增, MySQL 8.4)
└── nginx/反向代理      (新增, 用于暴露后端 API)
    域名: api-tasklist.ch-tools.org（新建子域名）
```

监控组件（Prometheus + Grafana + MySQL Exporter）暂不迁移，节省资源。如后续需要可再加回。

## 迁移步骤

### 第一阶段：代码改造（在本地完成）

#### 1.1 前端：支持外部 API 地址

修改 `frontend/src/api/index.js`：

```js
// 改前
const api = axios.create({
  baseURL: '/api',
  timeout: 60000
})

// 改后
const api = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || '/api',
  timeout: 60000
})
```

本地开发和 Docker 部署不受影响（走默认 `/api`），Cloudflare Pages 构建时通过环境变量指定后端地址。

#### 1.2 后端：添加 CORS 中间件

前端和后端不再同域，需要允许跨域。修改 `backend/app.py`：

```python
from fastapi.middleware.cors import CORSMiddleware

# 在 app 创建后添加
app.add_middleware(
    CORSMiddleware,
    allow_origins=[
        "https://tasklist.ch-tools.org",
        "http://localhost:3000",  # 本地开发
    ],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

#### 1.3 提交代码变更

将以上两处改动提交到 Git 仓库。

### 第二阶段：新服务器部署后端 + MySQL

#### 2.1 准备文件

在新服务器上克隆项目仓库，创建一个精简版 `docker-compose.prod.yml`，只包含后端和数据库：

```yaml
services:
  db:
    image: mysql:8.4
    container_name: tasklist-db
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
      MYSQL_DATABASE: ${MYSQL_DATABASE:-tasklist_db}
      MYSQL_USER: ${MYSQL_USER:-taskuser}
      MYSQL_PASSWORD: ${MYSQL_PASSWORD}
      TZ: Asia/Shanghai
    volumes:
      - mysql_data:/var/lib/mysql
    ports:
      - "127.0.0.1:3307:3306"  # 仅本机访问，避免和其他服务冲突
    networks:
      - tasklist_network
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost", "-u", "root", "-p${MYSQL_ROOT_PASSWORD}"]
      interval: 10s
      timeout: 5s
      retries: 5
    command: --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci --default-time-zone='+08:00'
    deploy:
      resources:
        limits:
          memory: 512M

  backend:
    build:
      context: .
      dockerfile: Dockerfile.backend
    container_name: tasklist-backend
    restart: always
    environment:
      DATABASE_URL: mysql+pymysql://${MYSQL_USER:-taskuser}:${MYSQL_PASSWORD}@db:3306/${MYSQL_DATABASE:-tasklist_db}?charset=utf8mb4
      SECRET_KEY: ${SECRET_KEY}
      AI_API_KEY: ${AI_API_KEY}
      AI_BASE_URL: ${AI_BASE_URL:-https://api.deepseek.com}
      AI_MODEL: ${AI_MODEL:-deepseek-chat}
      PYTHONUNBUFFERED: 1
      TZ: Asia/Shanghai
    depends_on:
      db:
        condition: service_healthy
    volumes:
      - ./logs/ai:/app/logs
    ports:
      - "127.0.0.1:8000:8000"  # 仅本机访问，由 Nginx 反代
    networks:
      - tasklist_network
    deploy:
      resources:
        limits:
          memory: 384M

volumes:
  mysql_data:
    name: tasklist_mysql_data

networks:
  tasklist_network:
    name: tasklist_network
    driver: bridge
```

#### 2.2 创建 .env 文件

从旧服务器复制 `.env` 文件到新服务器，确保密码、SECRET_KEY、AI_API_KEY 等配置一致。

#### 2.3 配置 Nginx 反向代理

在新服务器上配置 Nginx（如果已有 Nginx 则添加 server block，否则新建）：

```nginx
server {
    listen 80;
    server_name api-tasklist.ch-tools.org;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # AI 功能需要较长超时
        proxy_connect_timeout 120s;
        proxy_send_timeout 120s;
        proxy_read_timeout 120s;
    }
}
```

SSL 由 Cloudflare 代理处理（Cloudflare Proxy 模式），服务器端只需监听 80 即可。

#### 2.4 迁移数据库

从旧服务器导出数据库：

```bash
# 旧服务器
docker compose exec db mysqldump -u root -p tasklist_db > tasklist_backup.sql
```

传输到新服务器后导入：

```bash
# 新服务器：先启动数据库
docker compose -f docker-compose.prod.yml up -d db

# 等待 MySQL 就绪后导入
docker compose -f docker-compose.prod.yml exec -T db mysql -u root -p tasklist_db < tasklist_backup.sql
```

#### 2.5 启动后端

```bash
docker compose -f docker-compose.prod.yml up -d
```

验证后端正常：

```bash
curl http://127.0.0.1:8000/docs
```

### 第三阶段：Cloudflare 配置

#### 3.1 创建 API 子域名

在 Cloudflare DNS 中添加：

| 类型 | 名称 | 内容 | 代理 |
|------|------|------|------|
| A | api-tasklist | 新服务器 IP | 开启（橙色云朵） |

开启代理后 Cloudflare 自动提供 HTTPS，服务器端无需配置 SSL 证书。

#### 3.2 部署前端到 Cloudflare Pages

1. 登录 Cloudflare Dashboard → Pages → 创建项目
2. 连接 Git 仓库
3. 构建配置：
   - 框架预设: `Vue`（或不选，手动配置）
   - 构建命令: `npm run build`
   - 构建输出目录: `dist`
   - 根目录: `frontend`
4. 环境变量：
   - `VITE_API_BASE_URL` = `https://api-tasklist.ch-tools.org/api`
   - `NODE_VERSION` = `18`

#### 3.3 绑定自定义域名

在 Pages 项目设置中添加自定义域名 `tasklist.ch-tools.org`。Cloudflare 会自动更新 DNS 记录指向 Pages。

#### 3.4 配置 SPA 路由

在 `frontend/public/` 目录下创建 `_redirects` 文件，处理 SPA 路由刷新 404 问题：

```
/*  /index.html  200
```

## 注意事项

1. **数据库密码和 SECRET_KEY 必须和旧服务器一致**，否则已有用户的密码和 token 会失效
2. **AI_API_KEY 记得迁移**，否则算命/BMI/菜单识别功能不可用
3. **Grafana 暂时不可用**，如需恢复监控后续再加
4. **文件上传跨域**：菜单图片上传是 multipart/form-data，CORS 中间件已覆盖，但需实际测试确认
5. **Cloudflare Pages 每次 git push 自动构建部署**，后续前端更新无需手动操作
6. **后端更新**仍需 SSH 到新服务器手动 `git pull && docker compose -f docker-compose.prod.yml up -d --build`
